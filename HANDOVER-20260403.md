> 日期：2026-04-03（后续以 README / CHANGELOG 为准）

## 一、产品名称

- **中文**：码流  
- **英文 / 技术名**：CodeFlow  
- **对外完整写法**：码流（CodeFlow）  
- **主标语**：指令成流，智能随行  
- **副标语**：手机驭 AI，指令达团队  

PWA 文案与版本以 **`web/pwa/`** 为主源（`config.js`、`manifest.json`、`index.html`）；根目录同名文件为便于静态托管的同步副本。对外仓库：**[codeflow-pwa](https://github.com/joinwell52-AI/codeflow-pwa)**，Pages：**https://joinwell52-ai.github.io/codeflow-pwa/**

## 二、当前状态（摘要）

- 架构：**Cursor MCP 插件** + **PC Desktop（bridgeflow-nudger）** + **手机 PWA** + **WebSocket 中继**。  
- 插件目录名：**`codeflow-plugin/`**（历史文档若写 `bridgeflow-plugin`，以磁盘实际目录为准）。  
- 桌面产物：**`CodeFlow-Desktop.exe`**（见 `bridgeflow-nudger/pack.cmd`）。

## 三、项目文件结构（关键路径）

```
D:\BridgeFlow\
├── codeflow-plugin/              ← Cursor MCP 插件
│   ├── .cursor-plugin/plugin.json
│   ├── scripts/mcp_server.py
│   └── README.md
├── bridgeflow-nudger/            ← PC Desktop（目录名历史遗留；产品名：码流 Desktop）
├── web/pwa/                      ← PWA 主源
├── server/relay/                 ← 本地联调中继
└── README.md / CHANGELOG.md
```

### 插件 Symlink（示例）

```
C:\Users\Administrator\.cursor\plugins\local\codeflow → D:\BridgeFlow\codeflow-plugin
```

（若仍为 `bridgeflow` 链接名，仅为历史习惯，需指向 `codeflow-plugin` 目录。）

## 四、中继与配置

- 默认 WebSocket 路径：**`/codeflow/ws/`**（客户端 `relayUrl` 须与网关一致）。  
- 团队配置优先：**`docs/agents/codeflow.json`**（兼容 **`bridgeflow.json`**）。

以 `codeflow-plugin/scripts/mcp_server.py` 为准；具体 Tools 列表见插件内 `README.md`。
