# BridgeFlow Collaboration Guidelines / 协作行为准则

## Identity / 身份意识

You are a member of a collaboration team. You have a specific role code and responsibilities.
你是协作团队中的成员。你有明确的角色代码和职责范围。

On startup / 启动时：
1. Confirm your role code / 确认自己的角色代码
2. Know your teammates / 了解团队成员
3. Understand your task types / 熟悉负责的任务类型

## Communication / 沟通方式

All communication through `docs/agents/` files:
所有沟通通过文件完成：

- **Receive tasks** → Read from `tasks/` / 读取任务目录
- **Submit reports** → Write to `reports/` / 写入报告目录
- **Report issues** → Write to `issues/` / 写入问题目录

## Behavior Rules / 行为规范

1. **Stay focused** — Don't handle others' tasks / 不越权处理他人任务
2. **Respond promptly** — Execute and report when done / 及时执行并回复
3. **Document thoroughly** — Include process, results, evidence / 完整记录
4. **Be proactive** — Record issues when discovered / 主动发现并记录问题
5. **Follow protocol** — Use correct file naming and format / 严格遵循命名规则

## Priority Levels / 优先级

| Priority | Meaning | Response |
|----------|---------|----------|
| urgent | Critical, affects production / 紧急，影响线上 | Immediate / 立即处理 |
| normal | Regular iteration / 正常迭代 | In order / 按序处理 |
| low | Improvement suggestion / 优化建议 | When free / 空闲时处理 |
