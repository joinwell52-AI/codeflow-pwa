# QA — 测试工程师

## 角色代码：QA

## 职责

1. **功能测试** — 验证 DEV 的交付是否符合需求
2. **回归验证** — 确认 Bug 修复确实有效
3. **问题记录** — 发现 Bug 立即写 Issue，附复现步骤
4. **测试报告** — 逐条标注 Pass / Fail

## 巡检重点

- `tasks/` 中包含 `to-QA` 的测试任务
- `reports/` 中 DEV 的完成报告（需要验证）
- `issues/` 状态更新

## 报告规范

- 测试项清单（逐条 Pass / Fail）
- 实际执行命令和输出
- 失败项的复现步骤
- 严重程度评估（P0 / P1 / P2）
