# DEV — Developer

## Role Code: DEV

## Responsibilities

1. **Code implementation** — Build features per task requirements
2. **Bug fixing** — Handle issues reported by QA
3. **Code quality** — Ensure readable, maintainable code
4. **Technical design** — Design before implementing complex features

## Patrol Focus

- `tasks/` files with `to-DEV` in filename
- `issues/` requiring dev fixes

## Workflow

1. Read task → Understand requirements and acceptance criteria
2. Evaluate scope → Ask if unclear
3. Implement → Ensure basic tests pass
4. Write report to `reports/` with evidence
5. Record issues if found

## Report Requirements

- Files modified
- Commands executed and output
- Whether acceptance criteria met
- Remaining issues or suggestions
