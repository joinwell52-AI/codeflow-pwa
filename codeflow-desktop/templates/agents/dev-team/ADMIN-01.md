# ADMIN-01 真人管理员

【角色】真人管理员，编号 `ADMIN-01`  
【项目】`码流（CodeFlow）`

`ADMIN-01` 不是 AI，也不是远程操控器，而是项目组中的真人成员入口。

本角色通过 `agent_bridge` 协议进入团队协作流。

## 核心职责

1. 通过手机端或网页端向 `PM-01` 发需求
2. 追问任务进度
3. 接收 `PM-01` 或团队回执
4. 打开 OSS / Markdown 链接查看详细结果

## 核心规则

### 1. 一条消息 = 一条任务文件

`ADMIN-01` 发出的每条文本，必须落成：

```text
TASK-YYYYMMDD-序号-ADMIN01-to-PM01.md
```

不允许只停留在聊天窗口里。

### 2. 不直接跨角色派单

第一阶段里，`ADMIN-01` 默认只把文本发给 `PM-01`。

也就是说：

- 可以：`ADMIN01 -> PM01`
- 暂不默认支持：`ADMIN01 -> DEV01`
- 暂不默认支持：`ADMIN01 -> OPS01`
- 暂不默认支持：`ADMIN01 -> QA01`

### 3. 回复也必须文件化

`PM-01` 或其它角色回给 `ADMIN-01` 的文本，也必须落成：

```text
TASK-YYYYMMDD-序号-PM01-to-ADMIN01.md
```

或：

```text
TASK-YYYYMMDD-序号-DEV01-to-ADMIN01.md
```

### 4. 手机端不做 GUI 操作

`ADMIN-01` 的职责是“文本协同”，不是“操控 Cursor 窗口”。

因此：

- 不负责点击标签
- 不负责 OCR 识别
- 不负责发键盘输入到桌面窗口
- 不直接参与 PC 巡检动作

## 标准工作流

### 发任务

1. `ADMIN-01` 在手机端输入文本
2. 文本通过中继到达桌面桥接器
3. 桌面桥接器写出 `TASK-*-ADMIN01-to-PM01.md`
4. `PM-01` 接收并处理

### 看回复

1. `PM-01` 写出 `TASK-*-PM01-to-ADMIN01.md`
2. 桌面桥接器扫描到该文件
3. 通过中继发送摘要给手机端
4. `ADMIN-01` 在手机端打开详情或链接

## 对 PM-01 的要求

`PM-01` 处理 `ADMIN-01` 发来的任务时，必须：

1. 明确回复是否已接单
2. 若需要拆任务，继续写 `PM01-to-DEV01 / OPS01 / QA01`
3. 当出现阶段结果时，及时回 `PM01-to-ADMIN01`
4. 不允许长期只在内部流转，不给 `ADMIN-01` 回执

## 适用场景

- 不在电脑前时，用手机加入协作
- 快速派发需求给 PM
- 随时查看项目推进
- 接收交付链接与阶段回执

## 第一阶段限制

- 暂不支持完整会话线程 UI
- 暂不支持多管理员并发冲突处理
- 暂不支持手机端直接查看全部团队内部细节
- 暂不支持远程桌面控制
