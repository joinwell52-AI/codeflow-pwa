## 二、激活 Tab 识别（代码已修，待你本机打包验证）

### 原问题简述

- 实测时 `get_active_tab_role()` 常返回空 → 误判；根因包括：**高 DPI 下 `pyautogui.screenshot(region=窗口矩形)` 与 PrintWindow/OCR 像素网格不一致**、Tab 带截到菜单栏等。

### 已在 `cursor_vision.py` 落地的修复

1. **不再用 pyautogui 截 Tab 带**：改用 `scan()` 已生成的 **`state.screenshot`（PrintWindow 全窗口图）**，按 `win` 尺寸比例 `sx/sy` 裁剪 **y≈55~105（逻辑）对应的像素行**，与 OCR 同源坐标系。
2. **亮度阈值**：由固定 `max_b - 4` 改为 **随列亮度极差 `span` 自适应**（`margin = clamp(4 + span*0.12, 3..14)`），避免暗色主题下整条过阈值。

### 你本地仍需做的验证

1. `bump` 版本并 **PyInstaller 打包**（见第四节）。
2. 跑一轮「实测」，检查 `%APPDATA%\CodeFlow\tab_debug.png`：**应主要是 Agent Tab 栏**，不应是大块菜单栏。
3. 若仍失败：把 `tab_debug.png` + 日志里 `[active_tab]` 行贴给下一任；可再微调 `TAB_MENU_SKIP` / `TAB_STRIP_H` 或 `margin` 公式。

**关键代码**：`cursor_vision.py` → `get_active_tab_role()`（约第 1001 行起）

## 五、禁止事项

| 禁止 | 原因 |
|------|------|
| 用 PowerShell Set-Content 写 Python/HTML 文件 | 编码破坏 UTF-8 中文 |
| 混用 Python 版本打包 | 依赖不兼容 |
| 改 build.spec 的 hiddenimports | 少一个就运行时 ImportError |
| 假设 Agent 侧栏在左侧 | 实际在右侧（x≈2676） |
| 假设没有 DPI 缩放 | 测试机是 200% 缩放，3000×2000 |
| 用 `max_b - 4` 固定阈值 | 不同场景亮度差异大，应动态调整 |
