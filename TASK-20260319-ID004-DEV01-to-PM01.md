## 三、本次修复清单

### 3.1 饼图缺失（核心问题）

**现象**：页面上饼图+柱状图都能正常显示，但 PDF 里始终没有饼图。

**根因**：前端 `_captureChart()` 方法只返回单张图的 base64。当页面有多个图表（饼图+柱状图）时，只截到最后一个（柱状图），饼图丢失。后续尝试 Canvas 拼接方案也不稳定。

**修复方案**：彻底改为**多图数组传输**：
1. 前端 `_captureChart()` → `_captureCharts()`：返回 `string[]`，用 `echarts.getDataURL()` 逐个截取每张图表
2. 前端新增 `chart_images` 字段，传 base64 数组给后端
3. 后端 `build_pdf()` 新增 `chart_images` 参数：
   - 单张图 → 居中显示（最大 300pt 宽 × 180pt 高）
   - 多张图 → 用 reportlab Table 横向排列，自动分配宽度
4. 兼容旧的 `chart_image_b64` 单图字段

**验证结果**：饼图+柱状图均出现在 PDF 中，横向排列，尺寸合理。

### 3.2 PDF 中文符号修复

| 问题 | 原因 | 修复 |
|------|------|------|
| `¥` 显示为方框 | SimHei 不含 U+00A5 | `_strip_special()` 替换为 `￥`(U+FFE5) |
| `**粗体**` 原样显示 | 未处理 Markdown 内联语法 | `_cell_text()` 转 `<b>` 标签 |
| emoji 显示为方框 | SimHei 无 emoji 字形 | `_strip_special()` 统一清理 |
| `[文字](链接)` 残留 | 未清理 Markdown 链接 | `_cell_text()` 提取纯文字 |

### 3.3 PDF 图表尺寸优化

- 单图：最大 300pt × 180pt，居中
- 多图：横向排列，每张图按比例分配宽度，最高 160pt
- 不再占满整页
