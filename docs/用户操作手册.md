## 一、产品简介

BridgeFlow 是一套**多 AI 角色团队的人机协作桥接工具**：

```
手机（你）   ──发任务──>   中继服务器   ──转发──>   PC（AI 团队）
            <──回复摘要──              <──推送──
```

- **手机端**：真人主控台（PWA，用手机浏览器打开，无需安装 App）
- **PC 端**：AI 团队执行机（Python 进程，运行在你的开发电脑上）
- **中继**：消息转发服务，已部署好，开箱即用

每一条任务都以标准 Markdown 文件落盘，不产生第二套聊天记录。

## 二、前置条件

### PC 端要求

| 条件 | 版本要求 | 说明 |
|------|----------|------|
| 操作系统 | Windows 10/11 / macOS 10.15+ / Linux（Ubuntu 20.04+） | 三平台均支持 |
| Python | **3.10 或以上** | 必须，低版本不支持 |
| Cursor | 任意版本 | 推荐安装，AI 角色在 Cursor 中运行 |
| 网络 | 可访问 `wss://ai.chedian.cc` | 中继服务器 |

### 检查 Python 版本

```powershell
# Windows
python --version
# 或
py --version

# macOS / Linux
python3 --version
```

输出应显示 `Python 3.10.x` 或更高。

> **Python 安装：** 前往 https://www.python.org/downloads/ 下载 3.10 或以上版本。  
> Windows 用户安装时勾选 **"Add Python to PATH"**。

### 手机端要求

| 条件 | 说明 |
|------|------|
| 手机浏览器 | Safari（iOS）/ Chrome（Android） |
| 网络 | 可访问 `https://joinwell52-ai.github.io` |
| 摄像头权限 | 扫码绑定时需要授权摄像头 |

## 四、初始化配置

在你的**项目工作目录**下执行初始化命令：

```powershell
cd D:\my-project          # 先进入你的项目目录
bridgeflow init
```

### 初始化会做什么？

| 步骤 | 说明 |
|------|------|
| 生成 `bridgeflow_config.json` | PC 端运行配置文件 |
| 生成唯一 `device_id` | 根据主机名自动生成 |
| 生成唯一 `machine_code` | 格式 `BF-XXXXXXXX`，用于手机扫码绑定 |
| 创建运行目录 `.bridgeflow/runtime/` | 存放运行状态文件 |
| 复制 Cursor 规则文件 | 将 5 个角色的 `.mdc` 文件复制到 `.cursor/rules/` |

### 输出示例

```
已生成配置：D:\my-project\bridgeflow_config.json
设备ID：my-pc
机器码：BF-A3F2C8D1
运行态目录：.bridgeflow/runtime
已安装 Cursor 规则：D:\my-project\.cursor\rules
```

### 指定中继参数（可选）

默认使用公共中继，如需使用私有房间（推荐）：

```powershell
bridgeflow init --room-key my-team-private-room-2026
```

完整参数：

```powershell
bridgeflow init \
  --project-root D:\my-project \
  --relay-url wss://ai.chedian.cc/bridgeflow/ws/ \
  --room-key my-private-room
```

> ⚠️ **重要**：如果多个团队共用同一 `room_key`，消息会互相可见。建议每个团队设置独一无二的 `room_key`。

## 五、启动运行

```powershell
cd D:\my-project
bridgeflow run
```

> **更简单的方式**：`bridgeflow init` 会在项目目录生成 **`启动BridgeFlow.bat`**（Windows）/ **`start_bridgeflow.sh`**（Mac/Linux），双击即可启动，不需要记命令。

### 启动后会发生什么？

1. 终端打印启动横幅（OS / Python / Cursor 检测结果）
2. 本地 HTTP 仪表盘启动（`http://localhost:18765`）
3. 浏览器自动打开仪表盘
4. PC 端连接中继，等待手机绑定和任务

### 启动横幅示例

```
====================================================
  BridgeFlow PC 执行机
====================================================
  操作系统  : Windows 10.0.19045
  Python    : 3.10.11  ✓
  Cursor    : ✓ 运行中
  设备ID    : my-pc
  机器码    : BF-A3F2C8D1
  中继      : wss://ai.chedian.cc/bridgeflow/ws/
====================================================

  仪表盘已启动：http://localhost:18765
  正在打开浏览器…
```

### 仪表盘内容

浏览器打开 `http://localhost:18765` 后，页面显示：

| 区域 | 内容 |
|------|------|
| 环境检测 | OS / Python / Cursor 状态 |
| 连接状态 | 中继是否已连接（绿色/红色） |
| **二维码** | 手机扫码绑定用，自动刷新 |
| 任务统计 | 今日任务数、回执数 |

### 指定配置文件运行

多个项目时，可以指定配置文件：

```powershell
bridgeflow run --config D:\project-a\bridgeflow_config.json
```

### 自动升级模式

发现新版时跳过确认、直接升级并重启（适合脚本/无人值守场景）：

```powershell
bridgeflow run --auto-upgrade
```

## 六、手机端 PWA 配置

### 6.1 打开 PWA

用手机浏览器（Safari / Chrome）打开：

```
https://joinwell52-ai.github.io/bridgeflow-pwa/
```

### 6.2 添加到主屏幕（推荐）

| 系统 | 操作 |
|------|------|
| iOS（Safari） | 点底部"分享"按钮 → "添加到主屏幕" |
| Android（Chrome） | 点右上角菜单 → "添加到主屏幕" |

添加后像 App 一样从主屏幕启动，有独立图标。

### 6.3 PWA 界面说明

```
┌─────────────────────────┐  ← 固定顶部
│ [Logo] BridgeFlow [●][我的] │     ● = 连接状态灯（绿=已连，红=未连）
├─────────────────────────┤
│ [今日任务][今日回复]      │  ← 看板统计
│ [进行中] [已完成]         │
├─────────────────────────┤  ↑ 以下可滚动
│ [PM] [DEV] [OPS] [QA]   │  ← 团队角色（点击切换任务列表）
├─────────────────────────┤
│ 任务清单                 │  ← 点任务展开详情
├─────────────────────────┤
│ 发送任务区               │  ← 选人员 → 输入文本 → 选优先级 → 发送
└─────────────────────────┘
点顶部"我的" → 独立全屏配置页
```

### 6.4 "我的"页配置

首次使用，进入顶部右角"我的"，确认以下设置：

| 设置项 | 说明 | 推荐值 |
|--------|------|--------|
| 中继地址 | WebSocket 服务器 | `wss://ai.chedian.cc/bridgeflow/ws/` |
| 房间 | 与 PC 端 `room_key` 一致 | 你在 `bridgeflow init` 时设置的值 |
| 自动连接 | 打开 PWA 是否自动连中继 | 开启 |

## 七、扫码绑定 PC 与手机

绑定后，手机和 PC 才能互相识别，发送的任务才会路由到正确的 PC。

### 步骤

```
1. PC 端运行 bridgeflow run
            ↓
2. 浏览器打开 localhost:18765，页面上显示二维码
            ↓
3. 手机打开 PWA → 顶部右角"我的" → 点"📷 扫码绑定 PC"
            ↓
4. 对准 PC 仪表盘上的二维码扫描
            ↓
5. 手机自动发送绑定请求 → PC 端自动确认
            ↓
6. 绑定完成，仪表盘连接状态变绿
```

### 绑定状态查询

```powershell
bridgeflow bind-status
```

输出示例：

```
绑定状态：bound
机器码：BF-A3F2C8D1
已绑定手机：我的iPhone  MOBILE-XXXX
绑定时间：2026-03-21 10:30:00
```

### 手动绑定（备用方案）

如果扫码失败，可以手动输入绑定码：

```powershell
# PC 端生成一次性绑定码
bridgeflow bind-code
# 输出：一次性绑定码：A1B2C3（10分钟有效）
```

在手机端"我的" → "手动输入绑定码"，输入 6 位码，完成绑定。

### 解绑

```powershell
bridgeflow unbind
```

## 八、发送任务与查看回复

### 从手机发送任务

1. 在 PWA 底部"发送任务区"
2. 选择接收角色（PM / DEV / OPS / QA）
3. 输入任务内容
4. 选择优先级（P0 紧急 / P1 高 / P2 中 / P3 低）
5. 点"发 送"

任务发送后，PC 端自动写入 `docs/agents/tasks/TASK-*.md` 文件。

### 从 PC 命令行发送任务（测试用）

```powershell
bridgeflow write-admin-task --text "请 PM 安排下一阶段任务计划"
```

### 查看任务列表

在 PWA 主页面，点顶部角色图标（PM / DEV / OPS / QA）切换查看对应角色的任务列表。

点击任意任务卡片，展开显示：
- 任务基本信息（编号、时间、优先级）
- 任务正文（Markdown 解析）
- 对话记录时间线

### 查看回复

AI 角色完成任务后，回执文件写入 `docs/agents/reports/`，PC 端自动推送 `reply_summary` 事件到手机端。手机端任务卡片上会显示最新回复摘要。

## 九、常用 CLI 命令

```powershell
# 初始化
bridgeflow init                                   # 生成配置
bridgeflow init --room-key my-room               # 指定房间
bridgeflow init --force                          # 强制重新生成

# 运行
bridgeflow run                                    # 启动（最常用）
bridgeflow run --config path/to/config.json      # 指定配置文件

# 绑定
bridgeflow bind-status                            # 查看绑定状态
bridgeflow bind-code                             # 生成一次性绑定码
bridgeflow unbind                                # 解绑

# 任务文件（测试/调试用）
bridgeflow write-admin-task --text "内容"
bridgeflow write-reply --sender PM01 --text "回复内容" --thread-key "xxx"

# 桌面控制
bridgeflow desktop-action --action focus_cursor  # 聚焦 Cursor 窗口
bridgeflow desktop-action --action inspect       # 巡检
bridgeflow desktop-action --action start_work    # 开工

# 环境诊断
bridgeflow doctor                                # 检查目录和环境
```

## 十、配置修改

### 修改房间（room_key）

```powershell
bridgeflow init --room-key new-room-name
```

**注意**：手机端"我的"中的房间也要同步改成相同值。

### 修改中继地址

```powershell
bridgeflow init --relay-url wss://your-relay/bridgeflow/ws/
```

### 直接编辑配置文件

用任意文本编辑器打开 `bridgeflow_config.json`，修改 `relay` 块：

```json
{
  "relay": {
    "url": "wss://ai.chedian.cc/bridgeflow/ws/",
    "room_key": "your-new-room-key"
  }
}
```

修改后**重启** `bridgeflow run` 生效。

## 十一、常见问题

### Q：`bridgeflow: command not found`

Python 的 Scripts 目录未在 PATH 中。解决：

```powershell
# Windows：添加到环境变量，或用完整路径
py -m bridgeflow run

# 或重新安装时确保 PATH 正确
pip install --user bridgeflow
```

### Q：`bridgeflow run` 报 `FileNotFoundError: bridgeflow_config.json`

未在包含配置文件的目录下运行。解决：

```powershell
cd D:\my-project             # 进入有 bridgeflow_config.json 的目录
bridgeflow run
# 或指定路径
bridgeflow run --config D:\my-project\bridgeflow_config.json
```

### Q：仪表盘显示"中继未连接"（红灯）

1. 检查网络是否能访问 `wss://ai.chedian.cc`
2. 确认 `bridgeflow_config.json` 中的 `relay.url` 正确
3. 尝试重启 `bridgeflow run`

### Q：手机扫码后没有反应

- 确认手机 PWA 中继地址与 PC 端 `room_key` 一致
- 确认手机已授权摄像头权限
- 确认 PC 端仪表盘（`localhost:18765`）可以正常显示二维码
- 尝试刷新仪表盘页面重新生成二维码

### Q：PWA 页面没有更新，还是旧版

Service Worker 缓存未更新。解决：

```
手机浏览器 → 设置 → 清除缓存 → 重新打开 PWA
```

或长按 PWA 图标 → 删除 → 重新从浏览器添加到主屏幕。

### Q：仪表盘显示"未检测到 Cursor"

仪表盘通过进程列表+注册表检测 Cursor。如果安装在非标准路径，可能误报。不影响任务收发功能，可忽略。

### Q：任务文件写在哪里？

默认写在 `bridgeflow init` 时的工作目录下：

```
docs/agents/tasks/TASK-*.md      ← 任务文件
docs/agents/reports/TASK-*.md   ← 回执文件
```

可在 `bridgeflow_config.json` 的 `project` 块中修改路径。

## 附录：任务文件格式

每条任务以标准 Markdown 文件存储，文件名格式：

```
TASK-YYYYMMDD-序号-发送方-to-接收方.md
```

示例文件头：

```yaml
---
protocol: agent_bridge
version: 1
kind: task
sender: ADMIN01
recipient: PM01
priority: P1
thread_key: 20260321-103000-ADMIN01-to-PM01
created_at: 2026-03-21 10:30:00
---

请 PM 安排下周的开发计划。
```

优先级说明：

| 优先级 | 含义 | 使用场景 |
|--------|------|----------|
| `P0` | 紧急 | 生产故障、需立即处理 |
| `P1` | 高 | 当日必须完成 |
| `P2` | 中 | 本周完成即可 |
| `P3` | 低 | 有空再做 |
