# BridgeFlow 产品设计说明

## 产品定位

`BridgeFlow` 不是远程桌面，也不是聊天软件。

它的定位是：

- **手机端**：真人 `ADMIN01` 的主控台 —— 发任务、看进度、看回复、绑定 PC
- **PC 端**：AI 团队的专用执行机 —— 收任务、写文件、巡检、桥接
- **中继**：轻量文本事件转发层，不落盘、不执行

核心原则：

> **团队中的每一次沟通，都可以落成标准文本文件。**

---

## 解决的问题

现有 Cursor 多角色协作存在两个断点：

1. 真人用户不在电脑前时，无法自然参与团队流程
2. 手机端如果只是聊天、不落文件，会形成第二套协议

`BridgeFlow` 解决：

- 让 `ADMIN01` 正式进入团队协议
- 让 `ADMIN01 <-> PM01` 的每条消息进入文件协议
- 让手机端看到的是**任务对**（任务+回复配对），而不是离散消息

## 手机端 PWA 设计（v1.6.0）

**访问地址：** https://joinwell52-ai.github.io/bridgeflow-pwa/

### 页面结构

```
┌──────────────────────────────┐  ← 固定顶部 Header（54px）
│  [logo] BridgeFlow  [●][我的] │     左：logo + 品牌名 + 宣传语
│                              │     右：连接灯（绿/红）+ 我的入口
├──────────────────────────────┤
│  [今日任务][今日回复]         │  ← 看板区（4个金属按钮）
│  [进行中] [已完成]            │
├──────────────────────────────┤  ↑ 以下为可上下滚动区域
│  [PM] [DEV] [OPS] [QA]       │  ← 团队区（4个角色图标，点击切换任务列表）
├──────────────────────────────┤     每个图标显示：角色名 + 状态（忙碌/等待/闲着）
│  任务清单                    │  ← 任务列表（默认 PM）
│  [任务卡片]                  │     点击任务卡片 → 展开详情+MD记录
│  [任务卡片]                  │     每条任务显示：标题/优先级/类型/状态/最新回复摘要
├──────────────────────────────┤
│  发送任务区                  │  ← 发送区
│  [人员选择]                  │     选人员 → 输入文本 → 选级别 → 发送
│  [文本输入框]                │     未选任务时：立为新任务
│  [P0/P1/P2/P3] [发 送]       │     已选任务时：追加到该任务记录
└──────────────────────────────┘
```

**"我的"页（全屏覆盖）：**

点顶部右角"我的"→ 弹出独立全屏页，包含：

| 模块 | 内容 |
|------|------|
| 语言切换 | 中文 / English |
| 绑定设备 | 当前设备 ID、绑定状态、已绑定 PC、扫码绑定 |
| 配置与版本 | 中继地址、房间、自动连接、版本号 |
| 执行机控制 | 聚焦 Cursor / 巡检 / 开工 / 查看日志 |
| 连接状态 | 最近连接/断开时间、重连状态 |
| 系统日志 | 可展开查看实时日志 |

### 视觉风格

- **金属拉丝背景**：银灰色金属渐变，带横向拉丝纹理条纹
- **内陷插槽感**：输入框 `inset` 阴影，营造凹槽质感
- **铆钉按钮感**：按钮上高光+下暗影，立体浮雕效果
- **电蓝高光**：`#2563eb / #3b82f6`，用于选中、连接成功、发送按钮
- **连接指示灯**：绿色=已连接（有绿色光晕），红色=未连接

### 配色体系

```css
--metal-bg:   银灰渐变背景
--metal-card: 卡片金属渐变（多层 rgba 叠加拉丝纹理）
--blue:       #2563eb（主蓝）
--blue-hi:    #3b82f6（亮蓝）
--text:       #111827（深色文字，保证可读性）
--text2:      #374151
--text3:      #6b7280（辅助文字）
```

## PC 端仪表盘设计

`bridgeflow run` 启动后，自动在本地开启 HTTP 服务并打开浏览器：

**地址：** `http://localhost:18765`

仪表盘显示：

| 区域 | 内容 |
|------|------|
| 环境检测 | OS / Python 版本 / Cursor 安装状态（含运行状态） |
| 连接状态 | 中继连接是否成功、设备 ID、房间名 |
| **二维码** | 手机扫描后自动绑定（内嵌中继地址+机器码+房间） |
| 任务统计 | 今日任务数、回执数 |

二维码内容格式：

```
bridgeflow://bind?machine_code=BF-XXXX&relay=wss://ai.chedian.cc/bridgeflow/ws/&room=bridgeflow-default&device_id=xxx
```

## 任务对（Task Pair）模型

手机端展示的不是离散消息，而是**任务对**：

```
任务对 = 一条任务文件 + 若干回复文件（相同 thread_key）

显示内容：
  任务标题
  优先级 | 任务类型 | 接收角色 | 当前状态
  最新回复摘要（如有）
```

点击任务对 → 展开详情：
- 基本信息（编号/文件/发起人/接收人/时间/状态/流转路径）
- 任务正文（MD 解析）
- 任务记录（对话气泡时间线）
- MD 原文

## 通信协议（中继事件）

| 事件名 | 方向 | 含义 |
|--------|------|------|
| `command_from_admin` | 手机→PC | 发送任务文本 |
| `task_event` | PC→手机 | 新任务通知 |
| `reply_summary` | PC→手机 | 回复摘要推送 |
| `request_dashboard` | 手机→PC | 拉取任务状态 |
| `dashboard_state` | PC→手机 | 返回任务状态 |
| `request_task_detail` | 手机→PC | 拉取单任务详情 |
| `task_detail` | PC→手机 | 返回任务详情 |
| `request_bind_code` | 手机→PC | 申请绑定 |
| `bind_state` | PC→手机 | 返回绑定状态 |
| `approve_bind` | PC确认 | 确认绑定 |
| `unbind_device` | 解绑 | 解除绑定 |
| `execute_desktop_action` | 手机→PC | 执行桌面动作 |
| `desktop_action_result` | PC→手机 | 动作执行结果 |

---

## 第一阶段不做的事

- 手机远程控制 Cursor 窗口（PC 端只能聚焦，不能操作内容）
- 手机端 OCR / 语音输入
- 手机端代替 PC 巡检
- 中继传输大文件
- 复杂账号系统与后台管理
- 非 MD 格式的富文本编辑
