## 中继集成方案

中继 WebSocket 路由集成在 `saige-ai` 的 FastAPI 应用中，路径为：

```
/bridgeflow/ws/
```

相关文件：

```
saige-ai-module/backend/app/api/bridge_relay.py   # BridgeFlow 中继路由
saige-ai-module/backend/app/main.py               # 包含 include_router(bridge_relay.router)
```

Nginx 代理配置（`ai.chedian.cc`）：

```nginx
location /bridgeflow/ws/ {
    proxy_pass http://127.0.0.1:8000/bridgeflow/ws/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_read_timeout 3600s;
}
```

## PC 端配置（已是默认值）

`bridgeflow init` 生成的配置默认已指向公网：

```json
{
  "relay": {
    "url": "wss://ai.chedian.cc/bridgeflow/ws/",
    "room_key": "bridgeflow-default"
  }
}
```

> **生产建议**：把 `room_key` 改成自己的随机值（如 `company-team-2026`），避免与他人共用同一中继房间。

修改方式：

```powershell
# 方式一：重新初始化并指定房间
bridgeflow init --room-key company-team-2026

# 方式二：直接编辑 bridgeflow_config.json 中的 room_key 字段
```

## 手机端 PWA 配置

PWA 的中继配置在 `web/pwa/config.js`：

```javascript
window.BRIDGEFLOW_CONFIG = {
  appVersion: "1.6.0",
  relayUrl:   "wss://ai.chedian.cc/bridgeflow/ws/",
  roomKey:    "bridgeflow-default",
  autoConnect: true,
  relayLabel:  "公网正式中继"
};
```

PWA 部署在 GitHub Pages，域名为：

```
https://joinwell52-ai.github.io/bridgeflow-pwa/
```

更新 PWA 的流程：

```powershell
# 1. 修改 web/pwa/ 目录下的源文件

# 2. 升版本号（必须升，否则 Service Worker 缓存不更新）
# 修改 web/pwa/config.js 中的 appVersion

# 3. 同步到 bridgeflow-pwa/ 目录并推送
cd BridgeFlow\bridgeflow-pwa
Copy-Item ..\web\pwa\index.html index.html -Force
Copy-Item ..\web\pwa\config.js config.js -Force
git add .
git -c core.hooksPath=/dev/null commit -m "update: 说明改了什么"
git push origin main
```

## 如果需要独立部署 Relay

如果不想依赖 saige-ai 后端，也可以独立运行 Relay：

### 本地独立 Relay

```powershell
cd BridgeFlow
py -3.10 server/relay/server.py
# 监听 ws://127.0.0.1:5252
```

### 远端独立部署

```powershell
cd BridgeFlow
py -3.10 .\scripts\deploy_public_relay.py
```

脚本执行步骤：

1. 上传 `server/relay/server.py` 到远端 `/opt/bridgeflow/server.py`
2. 在远端创建独立虚拟环境 `/opt/bridgeflow/venv`
3. 安装 `websockets`
4. 写入 `systemd` 服务文件 `bridgeflow-relay.service`
5. 修改 Nginx 站点文件，添加 `/bridgeflow/ws/` 代理
6. 重载 `systemd` 与 `nginx`

如果 Nginx 站点文件路径不是默认值：

```powershell
$env:BRIDGEFLOW_NGINX_SITE = "/etc/nginx/sites-enabled/your-site"
py -3.10 .\scripts\deploy_public_relay.py
```

独立 Relay 的中继地址示例（替换为真实域名）：

```
wss://your-domain.com/bridgeflow/ws/
```

## 中继边界（公网版与本地版相同）

| 属性 | 值 |
|------|------|
| 单条消息限制 | `8KB` |
| 默认限流 | 10 秒内最多 20 条 |
| 落盘 | ❌ 不落盘 |
| 执行代码 | ❌ 不执行 |
| 文件传输 | ❌ 不传文件 |
| 鉴权 | 基于 `room_key` 房间隔离（后续补 `shared_secret`） |
