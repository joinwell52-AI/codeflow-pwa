# 把 `codeflow-plugin` 打成可发布的 PyPI 包 `codeflow-mcp` 0.2.0

## 目标

让 `codeflow-plugin/` 下的 MCP 工具箱成为一个可以

```bash
uvx codeflow-mcp          # 零污染全局
pip install codeflow-mcp  # 传统方式
```

一行启动的标准 PyPI 包，**且不破坏** ADMIN 当前走 `scripts/mcp_server.py`
绝对路径的 Cursor 配置。

## 交付清单（必须全做）

### 1. 目录结构改造

最终形态：

```
codeflow-plugin/
├── pyproject.toml                        # 新增
├── README.md                             # 保持不动（README 改版留给后续子任务）
├── requirements.txt                      # 保持不动（开发者直接 pip 装也能跑）
├── src/                                  # 新增
│   └── codeflow_mcp/
│       ├── __init__.py                   # 新增：__version__ = "0.2.0"；re-export main
│       ├── __main__.py                   # 新增：from .server import main; main()
│       ├── server.py                     # 从 scripts/mcp_server.py 搬过来的完整代码
│       └── _data/
│           └── codeflow-core.mdc         # 与 rules/codeflow-core.mdc 字节一致
├── rules/
│   └── codeflow-core.mdc                 # 保持不动（开发者视角的权威副本）
└── scripts/
    └── mcp_server.py                     # 缩成 shim（见 §3）
```

### 2. `pyproject.toml` 要求

- 构建后端：`hatchling`（简单、不需额外配置文件）。
- `name = "codeflow-mcp"`，`version = "0.2.0"`。
- `requires-python = ">=3.10"`（和 `codeflow-project.mdc` 对齐）。
- `dependencies = ["fastmcp>=3.2.0", "websockets>=12.0"]`（锁定现有 requirements）。
- `[project.scripts] codeflow-mcp = "codeflow_mcp.server:main"`。
- `[project.urls]`：Homepage / Repository / Issues 都指向 Bridgeflow 仓库
  （GitHub 地址如果 ADMIN 未确认就指向本地仓库占位，OPS 发布前可补）。
- 作者：`authors = [{name = "CodeFlow Team"}]`；
  license：`"MIT"`（若仓库根部无冲突 LICENSE，暂用 MIT，OPS 阶段再确认）。
- 分类器（classifiers）：至少含 `Programming Language :: Python :: 3`、
  `Programming Language :: Python :: 3.10/3.11/3.12`、
  `Development Status :: 4 - Beta`、
  `Environment :: Console`、
  `Topic :: Software Development`。
- 长描述：`readme = "README.md"`；如 README 引用了仓库内相对路径图片，
  PyPI 不会渲染得完美，可接受（后续子任务再美化）。
- `[tool.hatch.build.targets.wheel] packages = ["src/codeflow_mcp"]`。
- 包数据：`[tool.hatch.build] include = ["src/codeflow_mcp/_data/*.mdc"]`
  或通过 `force-include` 保证 `_data/codeflow-core.mdc` 进 wheel。

### 3. `scripts/mcp_server.py` 向后兼容 shim

必须满足：

- ADMIN 当前 `mcp.json` 里 `command = "py"`, `args = ["-3.12", "D:\\Bridgeflow\\codeflow-plugin\\scripts\\mcp_server.py"]`
  不改也能继续启动。
- 不依赖 `pip install codeflow-mcp`——通过 `sys.path.insert(0, "<this>/../src")`
  让它能 import 到源码树里的 `codeflow_mcp.server`。

参考形态：

```python
"""Backwards-compat entrypoint for ADMIN's existing mcp.json that points at
scripts/mcp_server.py. Prefer `codeflow-mcp` CLI after `uvx codeflow-mcp` /
`pip install codeflow-mcp`.
"""
import os, sys
_HERE = os.path.dirname(os.path.abspath(__file__))
_SRC = os.path.normpath(os.path.join(_HERE, "..", "src"))
if os.path.isdir(_SRC) and _SRC not in sys.path:
    sys.path.insert(0, _SRC)

from codeflow_mcp.server import main  # noqa: E402

if __name__ == "__main__":
    main()
```

### 4. `server.py` 调整项

搬迁 + 三处改动：

- 文件末尾原 `if __name__ == "__main__": mcp.run()` 包成 `def main(): mcp.run()`，
  并保留 `if __name__ == "__main__": main()`。
- `_core_mdc_hash()` 的候选路径列表，在原两条（项目 `.cursor/rules/`、
  `scripts/../rules/`）之外，**增加第三条**：`importlib.resources.files(
  "codeflow_mcp") / "_data" / "codeflow-core.mdc"`，并对 `MultiplexedPath`
  做 `read_bytes()` 兼容（用 `importlib.resources.as_file` 落盘再读）。
- `init_project()` 最后追加一个块：如果 `PROJECT_DIR / ".cursor" / "rules" /
  "codeflow-core.mdc"` 不存在，则从 `importlib.resources.files("codeflow_mcp") / "_data"`
  复制过去；复制前先 `mkdir(parents=True, exist_ok=True)`；复制失败不阻断初始化，
  只在返回串里追加一行"（codeflow-core.mdc 未释放：<原因>）"。

## 约束

- `rules/codeflow-core.mdc` 和 `src/codeflow_mcp/_data/codeflow-core.mdc` **必须字节一致**
  （和 `codeflow-desktop/templates/rules/codeflow-core.mdc` 一样的同步纪律）。
  推荐做法：建一个小脚本或 hook，把 `rules/codeflow-core.mdc` 复制到
  `_data/`；本单里手工保证一次就行。
- 不动 `codeflow-plugin/requirements.txt`、`codeflow-plugin/mcp.json`、
  `codeflow-plugin/templates/*`、`codeflow-plugin/agents/*`。
- 不引入 `setup.py`/`setup.cfg`（项目规则：`pyproject.toml` 权威）。
- 不打包 `.cursor-plugin/`、`agents/`、`commands/`、`hooks/`、`skills/`、
  `templates/`、`rules/`（留给 Cursor 侧配置，不属于 PyPI 包职责）。
- 不自动上传 PyPI——那是 OPS 的活，**要 ADMIN 先放行**。
