**最低 7 个场景必填**：

| 代号 | 场景 | 关键验证点 |
|---|---|---|
| TS-5.1 | InboxWatcher 检测 add 事件 | handler 必触发 1 次，filename / sender / recipient 解析正确 |
| TS-5.2 | InboxWatcher 忽略非 TASK-* | REPORT-*.md / HANDOFF-*.md / .DS_Store / .gitkeep 全部 SKIP |
| TS-5.3 | InboxWatcher handler 异常隔离 | handler 抛错后下一个 add 事件仍能正常触发 handler |
| TS-5.4 | TaskParser 正常解析 | frontmatter + body 完整；顶层便利字段（task_id/sender/...）正确映射 |
| TS-5.5 | TaskParser 容忍无 frontmatter | 文件不以 `---` 开头 → 返回 `{ frontmatter: {}, body: 全文 }`；不抛 |
| TS-5.6 | TaskParser YAML 解析失败 | 文件是 `--- 然后 YAML 但有语法错误` → throw TaskParseError |
| TS-5.7 | StateHistoryWriter 第 1 次 append | 末尾出现完整 `\n---\n\n## state_history (auto-appended by runtime)\n\n- ...` 节 |
| TS-5.8 | StateHistoryWriter 第 N 次 append | 不重复 `## state_history` 标题，只追加 `- bullet` |
| TS-5.9 | StateHistoryWriter 文件不存在 | throw TaskFileNotFoundError |
| TS-5.10 | TaskDispatcher 正常 dispatch 链路 | startSession 被调用 1 次；state_history 出现 inbox→dispatched bullet |
| TS-5.11 | TaskDispatcher recipient 找不到 | state_history 出现 inbox→agent_not_found bullet；不调用 startSession |
| TS-5.12 | TaskDispatcher 同 agent reject_busy | 第二个 task 触发 reject_busy；state_history 出现 inbox→rejected_busy bullet |
| TS-5.13 | TaskDispatcher session 终结追加 | session ended/cancelled 后 state_history 出现 dispatched→ended/cancelled bullet |

**自由发挥**：QA 如果觉得需要更多场景，可以加（不少于 7 即可）。
