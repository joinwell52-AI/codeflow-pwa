# 本仓 fcop MCP 用户面命令对齐到 fcop-mcp 0.6.x

## 背景

ADMIN 5/9 10:21 在 chat 给出明确指示：

> 「本地的 fcop 已经抽离了，fcop 独立了；我们就是安装 fcop-mcp 啊」
> 「本地残留的旧的，已经是历史文件了，新的 D:\FCoP 已经独立了」

PM 在 5/9 10:19 的诊断（见上方 references）发现本仓 fcop MCP 用户面配置存在 v0.5 → v0.6 命令断档：

| 文件 | 现状 | 应当 |
|---|---|---|
| `codeflow-plugin/mcp.json` | `["fcop"]` | `["fcop-mcp"]` |
| `codeflow-plugin/scripts/install-fcop.ps1` | `@("fcop")` | `@("fcop-mcp")` |
| `codeflow-plugin/scripts/install-fcop.sh` | `["fcop-mcp"]`（git unstaged） | 已对，顺手 commit |
| `codeflow-plugin/README.md` 方式 A/B/D + 升级 | 多处用 `fcop` | 对齐到 `fcop-mcp` |

虽然 `fcop@0.6.1` 在 D:\FCoP 仓加了 compat shim，老用户跑 `uvx fcop` 不会爆炸（会看到迁移提示），但本仓自己的命令应该升到正版。

## 范围（in-scope）

### 必改

| 文件 | 改动 |
|---|---|
| `codeflow-plugin/mcp.json` | `args: ["fcop"]` → `args: ["fcop-mcp"]`；保留 server key `"fcop"`（这是 mcp.json 里的 server name，按本仓约定保留）|
| `codeflow-plugin/scripts/install-fcop.ps1` | `args = @("fcop")` → `args = @("fcop-mcp")`；顺手在文件头部 comment 加一行 `# Writes mcp.json with: uvx fcop-mcp (0.6.x; legacy "uvx fcop" is deprecated)` 与 .sh 对齐 |
| `codeflow-plugin/README.md` 方式 A | code block 里 `args: ["fcop"]` → `["fcop-mcp"]`；段落措辞 `uvx 拉 fcop` → `uvx 拉 fcop-mcp` |
| `codeflow-plugin/README.md` 方式 B | Cursor Deeplink 的 base64 重新编码：原 base64 = `{"command":"uvx","args":["fcop"]}` → 新 base64 = `{"command":"uvx","args":["fcop-mcp"]}` = `eyJjb21tYW5kIjoidXZ4IiwiYXJncyI6WyJmY29wLW1jcCJdfQ==` |
| `codeflow-plugin/README.md` 方式 D | `pip install fcop` → `pip install fcop-mcp`；`command: "fcop"` → `command: "fcop-mcp"`；「若 fcop 不在 PATH」→「若 fcop-mcp 不在 PATH」 |
| `codeflow-plugin/README.md` 升级章节 | `uv tool upgrade fcop` → `uv tool upgrade fcop-mcp`；`pip install -U fcop` → `pip install -U fcop-mcp` |

### 保留不动（因为这些不是用户面 MCP 命令配置）

| 文件 | 保留理由 |
|---|---|
| `codeflow-plugin/.cursor-plugin/plugin.json` `name: fcop` | 这是 Cursor plugin 自己的元数据，是 plugin 的*身份名*，不是 PyPI 包名 |
| `codeflow-plugin/pyproject.toml` `name = "fcop"` | 本仓 codeflow-plugin 子项目的 Python 包名是 `fcop`（按 §8.0 是 LEGACY 副本，但本次不改它的包名——那是 §8 v2 路线图里 v0.3 后才做的退役动作） |
| `codeflow-plugin/src/fcop/server.py` 内部 `from fcop import …` / `_pkg_version("fcop")` | 内部代码引用同包的 import，与用户面 MCP 命令无关 |
| `codeflow-plugin/src/fcop/_data/teams/index.json` `protocol: "fcop"` | 这是 FCoP 协议标识符，不是包名 |
| `codeflow-plugin/README.md` "本地开发"段 `python -m fcop` | 本地开发用 `pip install -e codeflow-plugin` 装的就是本仓 LEGACY 副本（Python 包名 = `fcop`），保留；但要加注解说明「本地开发是装本仓副本，运行时还是叫 fcop；正式安装走方式 A/B/D 装的是 D:\FCoP 发布的 fcop-mcp」 |

## 不做（out-of-scope）

- ❌ 改 `codeflow-plugin/pyproject.toml` 的 `name = "fcop"` —— 这会引发 PyPI 重命名 / breaking change，属于 §8 v2 路线图 v0.3 后的事，不在本次范围
- ❌ 删除 `codeflow-plugin/src/fcop/` 整个 LEGACY 副本目录 —— 同上
- ❌ 改任何与 v2 设计相关的文档（`docs/design/codeflow-v2-on-fcop-sdk.md` / `docs/codeflow-overview.md` 等已经在用 `fcop-mcp` 措辞）
- ❌ 改 user-level 全局 `${USERPROFILE}\.cursor\mcp.json` —— 那是 ADMIN 自己的环境，不是仓内文件，本任务管不着
