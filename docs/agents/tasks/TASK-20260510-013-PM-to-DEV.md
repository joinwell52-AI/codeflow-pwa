## §三 BUG-SDK-004：reviewer `verdict_parse_failed` 调查 + 修复

### 3.1 现象（DEV-012 §四 §4.1 + §六 S2）

REVIEW-01 SDK agent 启动 → 跑了 48KB stream（含 tool_call: glob）→ 末尾 0 字符 assistant message → `parseVerdict()` 拿 0 字 → `NeedsHumanGate` 兜底 → `decision=needs_human + trigger_reason=verdict_parse_failed`。

stderr 关键警告：
```
Ripgrep path not configured
```

### 3.2 PM 给 DEV 的调查路径（DEV 自选）

**第 1 阶段（调查，30 min）**：

| 路径 | 看哪些文件 | 假设 |
|---|---|---|
| H1：SDK ripgrep setup 缺失 | `node_modules/@cursor/sdk/dist/cjs/options.d.ts` 找 `configureRipgrepPath` / `setRipgrepPath` API；`packages/codeflow-runtime/src/registry/AgentSdkAdapter.ts` 看构造时是否需要 setup hook | reviewer tool_call 跑 ripgrep 失败 → reviewer 在 grep 搜任务文件途中 abort → 没输出 final VERDICT 行 |
| H2：reviewer 角色 system prompt 不要求 VERDICT 格式 | `kernel/skills/review/*.yaml` / `*.json` / `*.md` / `packages/codeflow-runtime/src/review/*` 找 reviewer prompt 模板 | reviewer 跑完了但 LLM 没被指令产出 VERDICT 行 |
| H3：parseVerdict() 容错过严 | `packages/codeflow-runtime/src/review/parseVerdict.ts` （或类似）看 regex 是否过严（仅匹配 `^VERDICT:` 而非也匹配 `VERDICT: ` 或 `## VERDICT`）| reviewer 输出了类似 `VERDICT: approved` 但格式有 markdown 装饰被 regex 漏掉 |

DEV 自选优先级（PM 倾向 H1 → H2 → H3，但 DEV 看代码再定）。

**第 2 阶段（修复，30 min）**：

按调查结果修。**最小改动原则**：

- 如 H1 真：在 `CursorSdkAdapter` 构造或 `RuntimeBootstrap` 启动时调 `Cursor.configureRipgrepPath?.(<path>)` 一次（如果 SDK 提供该 API 的话；如不存在就找 SDK 兜底 path）。可能需要 bundle ripgrep binary（`@vscode/ripgrep` 或同类）但**优先**先看 SDK 是否能 fallback。
