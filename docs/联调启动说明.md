# BridgeFlow 联调启动说明

## 当前可运行链路

```
手机 PWA  ──>  wss://ai.chedian.cc/bridgeflow/ws/  <──  PC 端 bridgeflow run
              （中继已集成到 saige-ai FastAPI 后端）
```

完整闭环：

1. PC 端 `bridgeflow run` 连接中继
2. 手机 PWA 连接同一中继房间
3. 手机发送任务文本 → PC 端写 `TASK-*.md`
4. PC 端扫描回执 → 手机端收到 `reply_summary`

---

## 一、PC 端安装

### PyPI 安装（推荐，生产环境）

```powershell
pip install bridgeflow
```

### 本地开发安装

```powershell
cd BridgeFlow
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
```

---

## 二、初始化

```powershell
bridgeflow init
```

执行后会：

1. 生成 `bridgeflow_config.json`（默认使用公网中继）
2. 生成 `device_id`（本机唯一标识）
3. 生成 `machine_code`（绑定用机器码，格式 `BF-XXXXXXXX`）
4. 创建运行态目录 `.bridgeflow/runtime/`
5. 复制 Cursor 规则文件到 `.cursor/rules/`（包含 5 个角色的 `.mdc` 文件）

生成的配置默认值：

```json
{
  "relay": {
    "url": "wss://ai.chedian.cc/bridgeflow/ws/",
    "room_key": "bridgeflow-default"
  }
}
```

> **生产环境建议**：把 `room_key` 改成自己的随机值，避免与他人共用同一房间。

---

## 三、启动

```powershell
bridgeflow run
```

启动后：

- 终端打印启动横幅（OS / Python / Cursor 检测结果）
- 自动在本地启动 HTTP 仪表盘（`localhost:18765`）
- 自动用默认浏览器打开仪表盘
- 连接公网中继，状态实时刷新

指定配置文件：

```powershell
bridgeflow run --config D:\BridgeFlow-PC\bridgeflow_config.json
```

> **注意**：运行 `bridgeflow run` 时，必须在包含 `bridgeflow_config.json` 的目录下，或通过 `--config` 指定完整路径。

---

## 四、手机端连接

1. 手机打开：https://joinwell52-ai.github.io/bridgeflow-pwa/
2. 点顶部右角"我的" → "📷 扫码绑定 PC"
3. 对准 PC 仪表盘（`localhost:18765`）上的二维码
4. 自动解析中继地址、机器码，发送绑定请求
5. 仪表盘连接成功后，手机端状态灯变绿

## 五、烟雾测试（本地联调）

如果不想连公网，可以本地起中继联调：

### 1. 启动本地中继

```powershell
cd BridgeFlow
py -3.10 server/relay/server.py
# 监听 ws://127.0.0.1:5252
```

### 2. 启动桌面桥接器（指向本地）

```powershell
bridgeflow run --config .\_smoke_test\bridgeflow_config.json
```

### 3. 写入测试任务

```powershell
bridgeflow write-admin-task --text "请 PM 帮我安排下一步任务"
```

写入位置：`_smoke_test/docs/agents/tasks/TASK-*.md`

### 4. 写入测试回执

```powershell
bridgeflow write-reply --config .\_smoke_test\bridgeflow_config.json --sender PM01 --text "已接单，开始拆解" --thread-key "demo-thread-001"
```

写入位置：`_smoke_test/docs/agents/reports/TASK-*.md`

手机端应收到 `reply_summary` 事件。

## 六、绑定操作

```powershell
# 查看当前绑定状态
bridgeflow bind-status

# PC 端生成一次性绑定码（手机端输入绑定码的备用方案）
bridgeflow bind-code

# PC 端确认手机绑定
bridgeflow approve-bind --code A1B2C3 --mobile-device-id MOBILE-XXXX --mobile-device-name "我的手机"

# 解除绑定
bridgeflow unbind
```

## 九、常见问题

**Q：`bridgeflow run` 报 `FileNotFoundError: bridgeflow_config.json`**

在包含 `bridgeflow_config.json` 的目录下运行，或指定 `--config` 路径：

```powershell
bridgeflow run --config D:\BridgeFlow-PC\bridgeflow_config.json
```

**Q：仪表盘显示"未检测到 Cursor"**

仪表盘通过进程列表 + 注册表 + 安装路径三路检测 Cursor。如果安装在非标准路径（如 `D:\Program Files\cursor\`），可能误报。实际不影响任务收发功能。

**Q：手机扫码后没有反应**

- 确认手机 PWA 已连接中继（状态灯变绿）
- 确认 PC 仪表盘可访问（`localhost:18765` 显示二维码）
- 确认网络可以访问 `wss://ai.chedian.cc`
