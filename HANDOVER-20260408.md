## 五、禁止事项（必读）

| 禁止 | 原因 |
|------|------|
| 打包统一用同一个 Python 版本，不要混用 | 当前用 3.10 打包，3.12 依赖也验证通过，可迁移，但迁移后要全量回归测试 |
| 不能随意改 `build.spec` 的 `hiddenimports` | 少一个就运行时 ImportError，EXE 启动黑屏 |
| 不能用 `Set-Content` 写 Python 文件 | PowerShell 默认 UTF-16 编码，会导致文件损坏、语法错误 |
| 不能直接在 EXE 二进制里搜明文字符串来验证打包 | PyInstaller 对 datas 做 zlib 压缩，直接搜必然 NOT FOUND，是误判 |
| 打包前必须先 `py_compile` 语法检查 | 缩进错误只在运行时报，不在打包时报，会导致功能静默失败 |
| 不能假设 Agent 侧栏在左侧 | 用户环境 Agent 侧栏在右侧，写死左侧会导致所有 OCR 定位失败 |
| 不能用 `max-height` 小值限制 `pfBody` | 内容多时被截断，下方区块视觉上"透出"造成布局乱 |
| 不能两处维护版本号（`main.py` 和 `web_panel.py`）而不同步 | 会导致面板显示版本和实际版本不一致 |
| 不能在 `_run()` 线程里用混合缩进 | Python 对缩进严格，混合会导致运行时静默崩溃，`running` 永远不被置 `False` |
| 不能忽略 Cursor Simple Browser 的缓存问题 | 它是 Electron Chromium，HTTP 头 `no-cache` 不可靠，必须用 URL 时间戳强制刷新 |
